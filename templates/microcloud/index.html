MicroCloud

打造简单、灵活、
轻量且功能丰富的開源云端架构

MicroCloud主要特性

  • 快速轻松部署集群
  • 自动安全更新和简化的升级
  • 支持运行虚拟机或系统容器以实现更高的密度
  • 支持远程认证、细化访问控制、项目和多用户设置功能
  • 简化大规模复制操作
  • 完全开源，并可通过Ubuntu Pro提供企业技术支持

部署的大小

为小集群和边缘部署而设计，MicroCloud 可从单节点开始运行，但为实现高可用性（HA，high availability）所需的最低默认数量至少使用 3 个节点。系统规模可灵活扩展，从单节点轻松扩展至多达 50 个节点的大型集群。

在几分钟内部署一个功能齐全且可扩展的云环境

自动化部署

MicroCloud 注重简单和自动化的设计。无论是在办公室后端还是在难以到达的远程边缘站点，MicroCloud 都能实现部署和运维的自动化，几乎无需派遣人员到现场维护。

自动更新确保安全性

即便是小规模的 MicroCloud 仍可能处理着大量数据，甚至运行敏感的工作负载。为了确保数据安全，及时应对安全性和漏洞更新是关键。MicroCloud 使用 snap 进行组件打包，以实在无缝更新的同时提供额外的隔离层

高效利用成本和资源

除了 Ubuntu Server 和 Desktop，MicroCloud 也可运行于专为边缘场景优化的轻量级操作系统 Ubuntu Core。结合 Ubuntu Core，用户可以构建极轻量化的云，不牺牲性能地高效利用资源，同时有效控制成本。

MicroCloud组件

由三个领先开源解决方案合并成一个轻量云
MicroCloud 将三个领先的开源解决方案合并成一个轻量云，提供企业可随时使用的功能：系统容器和虚拟机管理工具 LXD 实现计算和虚拟化；Ceph 提供块、文件系统和对象存储；OVN 用于网络。这些都是有频繁更新、得到良好支持的生产级解决方案，基于此三者出色开源工具，Canonical 能够交付构建简单的云平台 MicroCloud。
