设备恢复

现场设备维修成本高昂

现场手动修复IoT设备的成本通常会超出设备本身的成本。将操作员派遣到远程位置进行维护或干预，根据到现成的距离和设备的可访问性等不同因素可能会导致高昂的成本。而且，故障时间可能会导致额外的损失，如果是关键任务的设备，那么甚至更多。一个可靠的设备恢复系统对于避免这些成本和烦恼至关重要。

低接触的设备维护

现成设备的恢复应该是低接触式。物联网设备可能以成百上千或者成千上万地大规模部署。在这种规模下，维护运营成为了成本的重要驱动力。自动化和远程访问是低接触维护的关键。基本和重复的维护操作可以由操作系统自主执行。这使得设备操作员免于在大量设备上重复执行简单的维护任务，从而节省了大量时间。

为此，仅需要将复杂的维护操作上报给设备操作员。设备操作员可通过远程访问来执行与软件相关的复杂维护操作，避免了实际的成本，并且允许集中维护现场的所有设备——从而降低风险。

Ubuntu Core上设备恢复

ARM和x86

Ubuntu Core在ARM和x86 SoC上提供了一个稳健的设备恢复系统。设备可通过手动或者REST API远程恢复。

预认证的开发板免费

恢复已在认证的设备上可用，比如树莓派，无需额外成本。在未认证的开发板上，完整的认证Ubuntu Core需要适配费用。

如何工作

Ubuntu Core 20引入了一个安全的恢复系统。

恢复模式

Ubuntu Core提供了一个在启动时可激活或远程API调用的恢复模式。在恢复模式中，Ubuntu Core提供了一个为用户进入恢复选项的用户界面。然后，操作系统将继续获取配置文件和预置了配置文件关联的snap应用。

{{ image ( url="https://assets.ubuntu.com/v1/f880a3bd-Enterprise+support.svg", alt="", width="150", height="150", hi_def=True, loading="lazy" ) | safe }}

快照

在恢复系统中可以备份同一设备的多个快照（snapshot）。这些快照反映了设置配置和系统上已安装的snap应用的集合。设备操作员可以在恢复系统中创建，命名和保存此类快照，这样可以在需要时快速恢复所需的系统状态。可以通过专用UI手动进行恢复，也可以通过API调用远程地进行。
