什么是私有云？

私有云，例如OpenStack为企业/组织提供按需的计算、存储和其他可通过网络访问并专门为他们保留的资源——无论是在自己的数据中心还是通过第三方。

{{ image( url="https://assets.ubuntu.com/v1/f25ccac7-Private+Cloud.svg", alt="Private cloud", width="190", height="130", hi_def=True, loading="auto", ) | safe }}

优点：

  • 与公有云相比，私有云通常更具成本效益——尤其是在大规模和长期使用时。
  • 拥有私有云的企业/组织不会与其他企业/组织共享其托管资源，这意味着通常情况下，私有云能提供更好的性能。

缺点：

  • 私有云的部署和持续管理需要专门的团队和技术资源。

Canonical的Charmed OpenStack通过简化OpenStack的部署和提供自动化运维来解决这个问题，从而显着降低私有云的总拥有成本 (TCO)。

什么是公有云？

在公共云环境中，计算、存储和其他基础设施资源由外部提供商作为服务的形式提供。

{{ image( url="https://assets.ubuntu.com/v1/a3694a61-Public+Cloud.svg", alt="Public cloud", width="190", height="130", hi_def=True, loading="auto", ) | safe }}

优点：

  • 即刻访问基础设施即服务 (IaaS) 可加快企业/组织的产品和服务的上市时间。
  • 公有云提供了几乎无限制地扩展资源消耗的能力。

缺点：

  • 虽然公有云的总体成本看起来很有吸引力，但隐藏的附属成本通常使其成为云架构的昂贵选择。

主要的公有云提供商包括亚马逊（AWS）、微软Azure、Google Cloud Platform、甲骨文和IBM Cloud。

什么是混合云？

混合云架构是一种结合私有云的使用和一个或多个具有平台之间工作负载编排引擎的公有云服务的架构。

{{ image( url="https://assets.ubuntu.com/v1/45120edb-Hybrid+Cloud.svg", alt="Hybrid cloud", width="190", height="130", hi_def=True, loading="auto", ) | safe }}

优点：

  • 当以正确的方式执行时，混合（云）策略可提供最大的OpEx效率。
  • 混合云非常适合具有不同要求的工作负载。
  • 因为组织不依赖于单一云供应商，可防止供应商锁定。

缺点：

  • 跨各种云平台的工作负载编排可能是一个挑战。

用Canonical的Juju编排混合云工作负载

什么是多云？

多云架构是与混合云相关的架构。它是多个私有云、公有云或混合云的组合。

多云架构正迅速被视为企业事实上的云战略，因为81%的企业与多个云服务提供商合作。

{{ image( url="https://assets.ubuntu.com/v1/dd0465c8-Multi-Cloud.svg", alt="Multicloud", width="220", height="130", hi_def=True, loading="auto", ) | safe }}

优点：

  • 防止供应商锁定，因为不依赖于单一云服务供应商。
  • 在最佳环境中运行应用程序时，比在其他（云）模型中运行具有更大的灵活性。

缺点：

  • 如在此类环境中没有适当的工具来使工作负载运营变得容易，可能会给企业/组织的基础架构增加额外的复杂性。

什么是边缘云？

边缘云是在数据中心之外运行的微型云，其让计算和数据存储更接近所需的位置。

{{ image( url="https://assets.ubuntu.com/v1/f24913d5-Edge+Cloud.svg", alt="Edge cloud", width="210", height="140", hi_def=True, loading="auto", ) | safe }}

优点：

  • 借助边缘云，企业可以从带宽的增加和延迟的降低所带来的网络性能提升中受益。

缺点：

  • 边缘面临的最大挑战是增加投资和运营成本、最小化占地面积、运营自动化和安全性。
